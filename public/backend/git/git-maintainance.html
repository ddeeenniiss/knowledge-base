GitLab Maintenance

Sollte der Server neu gestertet werden oder Git aus einem anderen Grund nicht erreichbar sein kann es vorkommen, dass man den Service neu starten muss.

Dazu gibt es 2 wichtige Befehle welche über eine SSH-Verbindung ausgeführt werden können:

# Status anzeigen

gitlab-ctl status

# Service starten

gitlab-ctl start

# Service neu starten

gitlab-ctl restart

Sollte das nicht helfen, und die services nicht gestartet werden (fail wird angezeigt):

gitlab services fail
sudo systemctl enable gitlab-runsvdir.service
sudo systemctl start gitlab-runsvdir.service

Wenn im Frontend ein 502 Fehler angezeigt wird:

# Dauert etwa 3 Minuten

sudo gitlab-ctl restart gitlab-pages
